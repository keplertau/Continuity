# Contributing

Continuity is in an early draft phase.

The project should stay accessible to nontechnical users. Contributions are strongest when they help someone point an AI assistant at this repository and get value without installing anything.

Useful contribution areas:

- clearer setup instructions
- better Lite, Standard, or Project templates
- worked examples for common tools
- corrections to the specification
- provider-specific notes that preserve the provider-neutral model

Design constraints:

- Keep the default path lightweight.
- Do not require a database, server, command-line workflow, or paid platform for basic use.
- Preserve the register model: current context, preferences, decisions, open threads, evidence, and session notes should remain distinguishable.
- Mark inferred or provisional memory clearly.
- Do not erase unresolved questions just to make memory cleaner.

Open an issue or pull request with a short explanation of the problem being solved.
