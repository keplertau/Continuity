# Project Team Example

This is a fictional Project-mode Continuity setup for a small open-source documentation team.

It shows how the structure in `templates/project/` looks once it has real content in it: current context, derived briefings, individual decision entities, session summaries, evidence records, processes, open threads, a roadmap, and one low-authority fragment.

The example project is not real. It uses a fictional team, "Northstar Docs," preparing launch documentation for a fictional open-source command-line tool.

## What To Notice

- `continuity/index.md` tells future AI sessions what to read first.
- `continuity/current.md` stays short and current.
- `continuity/briefings/` contains compressed orientation derived from the lower layers.
- `continuity/decisions/` holds the source-of-truth decision records.
- `continuity/sessions/` summarizes what happened without replacing evidence.
- `continuity/evidence/` keeps source records and declares fidelity.
- `continuity/processes/` describes routines that actually run.
- `continuity/fragments/` preserves one useful idea without treating it as a decision.

Start with `continuity/index.md`.
