# Three-Lane Docs Navigation

## The Decision

The docs landing page uses three top-level reader paths: Start, Understand, Contribute.

## What It Constrains

- Navigation copy should not start from internal team roles such as "user," "maintainer," and "developer."
- New docs pages should be placed under the reader task they serve.
- The Quickstart belongs under Start; architecture and concepts belong under Understand; issue workflow and local setup belong under Contribute.

## Rationale

Task-based navigation lets readers find their next action before they know the project's internal vocabulary.

## Evidence

- `sessions/2026-05-11-docs-navigation-review.md`

## Briefing Guidance

- Tier 1 inclusion: yes.
- Briefing form: docs navigation uses Start, Understand, Contribute as the three reader paths.
- Retirement condition: only if usability testing shows readers cannot find core tasks through this structure.
